void update_error_LED ( ErrorCode  error)

writes LEDS to indicate errors as described above

If an error during initialization was detected, some combination of the blue, green, orange, and red LEDs will be on, The combination of all 4 lights will tell you what the error is

NONE => Nothing is wrong BLUE => LOW G could not init GREEN => Could not begin SD card ORANGE => Could not set LOW G range RED => LOW G Output Data Rate Low Pass Filter could not be set BLUE, GREEN => HIGH G could not init BLUE, ORANGE => HIGH G could not update Data Rate BLUE, RED => MAGNETOMETER could not be init GREEN, ORANGE => GYRO could not init GREEN, RED => GPS could not init ORANGE, RED => CONTINUITY could not init BLUE, GREEN, ORANGE => BNO could not connect BLUE, GREEN, RED => BNO could not init BLUE, ORANGE, RED => GREEN, ORANGE, RED => BLUE, GREEN, ORANGE, RED => Default, somehow the function was called with a different error, should never happen
